## вңЁ What is Campus Buzz?

**Campus Buzz** is a real-time campus coordination web platform built exclusively for **NIT Raipur (NITRR)** students, clubs, and administrators. It solves the fragmentation of campus life by unifying social posting, event management, live chat, anonymous complaints, and club announcements вҖ” all in one place, accessible only with a verified `@nitrr.ac.in` email.

> _"One platform. Every beat of campus life."_

## вҡЎ Quick Start

### Prerequisites

```bash
node --version   # >= 18.0.0
npm --version    # >= 9.0.0
```

You also need a **MongoDB Atlas** cluster and a **Gmail App Password** for email.

### 1пёҸвғЈ Clone

```bash
git clone https://github.com/MAYANK-2109/CAMPUS-BUZZ.git
cd CAMPUS-BUZZ
```

### 2пёҸвғЈ Backend Setup

```bash
cd backend
npm install
```

Create `backend/.env` (see [Environment Variables](#-environment-variables) below), then:

```bash
npm run dev          # Development with hot-reload (nodemon)
# or
npm start            # Production
```

Server starts at **http://localhost:5000**

### 3пёҸвғЈ Frontend Setup

```bash
cd frontend
npm install
```

Create `frontend/.env`, then:

```bash
npm start            # Starts CRA dev server at http://localhost:3000
```

The frontend proxies all `/api/*` requests to `http://localhost:5000` automatically.

## рҹ”Ң Socket Events

Campus Buzz uses **two types of Socket.io rooms**:

### Post-Linked Rooms (Chat per post)
```
Client вҶ’ Server                     Server вҶ’ Client
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ               в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
joinRoom     { postId }         вҶ’   joinedRoom   { roomId, history }
sendMessage  { postId, text }   вҶ’   newMessage   { payload }
closeRoom    { postId }         вҶ’   roomClosed   { postId, closedBy }
cabLocation  { postId,lat,lng } вҶ’   cabLocationUpdate { lat, lng, sharedBy }
```

### Global Hub Rooms
```
Client вҶ’ Server                     Server вҶ’ Client
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ               в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
joinGlobalRoom  { roomId }      вҶ’   globalJoined       { roomId, history, room, onlineCount }
sendGlobalMsg   { roomId,text } вҶ’   globalMessage      { payload }
leaveGlobalRoom { roomId }      вҶ’   globalUserLeft     { userId, displayName }
closeGlobalRoom { roomId }      вҶ’   globalRoomClosed   { roomId, closedBy }
                                вҶ’   onlineUsersUpdate  { roomId, users[] }
                                вҶ’   roomsUpdated       (broadcast)
```

> All socket connections require a valid **JWT** in `socket.handshake.auth.token`.

## вҸ° Background Jobs

A single cron job runs **every 5 minutes** (`*/5 * * * *`):

| Task | Logic |
|------|-------|
| **Pre-expiry warning** | Finds `#foodsplit` / `#cabsplit` posts expiring in вүӨ35 min вҶ’ sends `expiry_warning` notification to author (once per post via `expiryWarned` flag) |
| **Post expiry** | Soft-deletes posts where `expiresAt < now` & `isActive = true` вҶ’ also deactivates linked ChatRoom |
| **Idle global room cleanup** | Closes global hub rooms with `lastMessageAt < 2 hours ago` |
| **Announcement expiry** | Soft-deletes announcements where `expiresAt < now` |

## рҹҡҖ Deployment

### Recommended Stack

| Service | Purpose |
|---------|---------|
| [**Render**](https://render.com) | Backend Node.js server (free tier) |
| [**Vercel**](https://vercel.com) | Frontend React app (free tier) |
| [**MongoDB Atlas**](https://www.mongodb.com/atlas) | Database (M0 free cluster) |

### Render (Backend)

1. Connect your GitHub repo вҶ’ **Web Service**
2. Build command: `npm install`
3. Start command: `npm start`
4. Set all environment variables from [`backend/.env`](#-environment-variables)
5. Set `NODE_ENV=production` and `CLIENT_URL=https://your-vercel-app.vercel.app`

### Vercel (Frontend)

1. Connect your GitHub repo вҶ’ **Framework Preset: Create React App**
2. Root directory: `frontend`
3. Set environment variables:
   - `REACT_APP_API_URL=https://your-render-backend.onrender.com/api`
   - `REACT_APP_SOCKET_URL=https://your-render-backend.onrender.com`

> **DNS Note:** The backend uses Google DNS (`8.8.8.8`) to resolve MongoDB Atlas SRV records, which may be blocked on restrictive college networks.
